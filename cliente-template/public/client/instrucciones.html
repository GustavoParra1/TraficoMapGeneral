Solución: Mapa Se Tilda/Congela

Guía paso a paso para debuggear el problema

Paso 1: Verificar que los datos se guardaron

Ir a Panel de Diagnóstico

Este panel te mostrará exactamente qué datos están en Firestore:

Abrir Diagnóstico
Importante: Espera a que cargue el panel. Verás un reporte como:
✅ Cliente encontrado: user123 📊 siniestros: 500 documentos 📊 cameras_privadas: 500 documentos 📊 semaforos: 200 documentos

Si ves "0 documentos":

Problema: Los datos NO se guardaron en Firestore.

Solución:

  1. Vuelve a Panel de Carga
  2. Abre DevTools (F12) → Console
  3. Arrastra el CSV nuevamente
  4. Busca errores en la consola (líneas rojas)
  5. Envía screenshot de errores

Paso 2: Verificar Permisos de Firestore

Error típico: "permission-denied"

En Console (F12) ves algo como:

❌ Error: PERMISSION_DENIED: Missing write permission
¿Por qué? Las reglas de Firestore NO permiten escribir datos

Solución:

  1. Ir a Firebase Console: https://console.firebase.google.com
  2. Seleccionar tu proyecto (laplatamaps-52a3b)
  3. Ir a: Firestore Database → Rules
  4. Reemplazar todo por:
    rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /clientes/{clientId}/{document=**} { allow read, write: if request.auth != null; } } }
  5. Click: Publish

Paso 3: ¿Por qué se tilda el mapa?

✅ BUENA NOTICIA: Ya lo arreglamos. El mapa ahora:
  • Carga datos en lotes de 50 (no 500 de una vez)
  • Muestra progreso en tiempo real
  • Le da tiempo al navegador para renderizar

Cómo ver el progreso:

Abre DevTools (F12) y ve la Console:

📍 Progreso: 50/500 📍 Progreso: 100/500 📍 Progreso: 150/500 ✅ 500 siniestros cargados

Paso 4: Probar el Mapa Actual

Ir a la página del Mapa

Abrir Mapa

En la Console (F12) deberías ver:

✅ Inicializando mapa para cliente: user123 🔄 Cargando siniestros desde: clientes/user123/siniestros 📊 Total de siniestros: 500, cargando en lotes... 📍 Progreso: 50/500 📍 Progreso: 100/500 ... ✅ 500 siniestros cargados
Paciencia: Cargar 500+ datos puede tardar 10-30 segundos. Es normal.

Paso 5: Diagnóstico Avanzado

¿Si aún no funciona?

Abre DevTools (F12) → Console y ejecuta:

firebase.auth().currentUser?.email

Debería mostrarte tu email. Si muestra undefined, no estás autenticado.

Ver estructura de tu cliente:

En Console ejecuta:

firebase.firestore() .collection('clientes') .get() .then(snap => { snap.forEach(doc => { console.log('Cliente:', doc.id, doc.data()); }) })

Ver datos cargados en una colección:

firebase.firestore() .collection('clientes/user123/siniestros') .limit(1) .get() .then(snap => { snap.forEach(doc => { console.log('Siniestro datos:', doc.data()); }) })
